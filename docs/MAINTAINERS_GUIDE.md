# 🧭 Maintainers Guide  
### Responsibilities, expectations, and best practices for Stays maintainers

Maintainers play a crucial role in ensuring the quality, stability, and growth of the Stays project. This guide outlines how to fulfill that role effectively.

---

## 🧩 Core Responsibilities

### **1. Reviewing Pull Requests**
- Ensure code quality  
- Check for tests and documentation  
- Verify architectural alignment  
- Provide constructive feedback  
- Merge when ready  

### **2. Managing Issues**
- Triage new issues  
- Apply labels  
- Close duplicates  
- Request clarification when needed  
- Prioritize based on roadmap  

### **3. Maintaining Code Quality**
- Enforce style guides  
- Identify refactoring opportunities  
- Ensure consistency across platforms  
- Promote best practices  

### **4. Supporting Contributors**
- Answer questions  
- Offer guidance  
- Encourage participation  
- Mentor newcomers  

### **5. Protecting the Project**
- Enforce the Code of Conduct  
- Respond to security reports  
- Prevent harmful changes  
- Maintain a positive environment  

---

## 🧪 PR Review Checklist

Before approving a PR:

- [ ] Code is readable and maintainable  
- [ ] Tests are included or updated  
- [ ] Documentation is updated  
- [ ] No secrets or sensitive data  
- [ ] No breaking changes without discussion  
- [ ] Follows project architecture  
- [ ] CI checks pass  

---

## 🛠️ Merging Guidelines

- Use **squash merge** for small or messy PRs  
- Use **merge commit** for large, well‑structured PRs  
- Avoid rebasing public branches  
- Ensure commit messages are meaningful  

---

## 🧭 Release Management

Maintainers may:

- Tag releases  
- Update changelogs  
- Coordinate with project owners  
- Communicate breaking changes  

---

## 🧘 Handling Conflict

If disagreements arise:

- Keep discussions respectful  
- Seek clarification  
- Focus on shared goals  
- Escalate to project owners if needed  

---

## 🌱 Growing as a Maintainer

We encourage maintainers to:

- Learn new technologies  
- Explore unfamiliar parts of the codebase  
- Share knowledge  
- Propose improvements  
- Lead initiatives  

---

## ❤️ Thank You

Maintainers are the backbone of the Stays project.  
Your leadership, care, and expertise make this community thrive.
