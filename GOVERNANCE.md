# 🏛️ Governance Model  
### How decisions are made in the Stays project

The Stays project is open, collaborative, and community‑driven. This document explains how decisions are made, who maintains what, and how contributors can grow into leadership roles.

---

## 👥 Roles

### **1. Project Owners**
Responsible for:

- Vision and long‑term direction  
- Approving major architectural changes  
- Managing releases  
- Overseeing security and compliance  

### **2. Maintainers**
Experienced contributors who:

- Review and merge pull requests  
- Manage issues and labels  
- Enforce coding standards  
- Mentor new contributors  
- Participate in roadmap planning  

### **3. Contributors**
Anyone who submits:

- Code  
- Documentation  
- Bug reports  
- Design assets  
- Testing support  

Contributors can become maintainers through consistent, high‑quality participation.

---

## 🧭 Decision‑Making Process

### **Minor Changes**
Handled by maintainers:

- Bug fixes  
- Small features  
- Documentation updates  
- Refactoring  

### **Major Changes**
Require discussion and approval from project owners:

- Architectural changes  
- API versioning  
- Database schema changes  
- New platform support  
- Deprecation of features  

Major changes should be proposed via:

- GitHub Issue  
- RFC (Request for Comments) in `/docs/rfcs`  

---

## 🗳️ Consensus Model

We use **“lazy consensus”**:

- If no objections are raised within a reasonable time, the change is accepted  
- If objections arise, maintainers discuss and refine  
- Project owners make final decisions when consensus cannot be reached  

---

## 📅 Roadmap Planning

Roadmap updates occur:

- Quarterly (major planning)  
- Monthly (minor adjustments)  

Community feedback is always welcome.

---

## 🚨 Conflict Resolution

If disagreements occur:

1. Discuss respectfully in the issue or PR  
2. Escalate to maintainers  
3. If unresolved, project owners make the final call  

Harassment or disrespectful behavior is not tolerated.

---

## 🌱 Becoming a Maintainer

We look for contributors who:

- Submit high‑quality PRs  
- Participate in reviews  
- Demonstrate reliability  
- Communicate clearly  
- Show alignment with project values  

Maintainers are invited by project owners.
